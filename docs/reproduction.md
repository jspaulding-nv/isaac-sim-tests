# Reproduction Guide

## 1. Host Prerequisites

Confirm Docker can see the GPU through CDI:

```bash
nvidia-smi
nvidia-ctk cdi list
docker run --rm --device=nvidia.com/gpu=all \
  nvcr.io/nvidia/cuda:13.0.0-base-ubuntu22.04 nvidia-smi
```

Authenticate to NGC if your environment requires it, then pull the pinned
images:

```bash
docker pull nvcr.io/nvidia/isaac-sim:6.0.1
docker pull nvcr.io/nvidia/isaac-lab:3.0.0-beta2-post1
```

Running an NVIDIA image with `ACCEPT_EULA=Y` indicates acceptance of the image
license. Review the applicable license before continuing.

## 2. Configure Streaming

```bash
cp deploy/.env.example deploy/.env
${EDITOR:-vi} deploy/.env
```

Set `ISAACSIM_HOST` to an address reachable from the browser. Keep the signal,
media, and viewer ports on a trusted network. Example firewall requirements:

| Purpose | Default |
|---|---|
| Web viewer | 8210/TCP |
| WebRTC signaling | 49200/TCP |
| WebRTC media | 48100/UDP |

Start and inspect:

```bash
./deploy/start.sh
./deploy/status.sh
docker compose --env-file deploy/.env -p isaacsim-tests \
  -f deploy/docker-compose.yml logs -f isaac-sim
```

The first launch can spend several minutes compiling shaders.

## 3. Low-Cost Gates

Run these before the full matrix:

```bash
./scripts/collect_inventory.sh
./scripts/run_compatibility.sh

BODIES=128 STEPS=120 ./scripts/run_gpu_physx_smoke.sh

NUM_ENVS=256 MAX_ITERATIONS=2 \
  ./scripts/run_isaac_lab_training.sh

PHYSICS_PRESET=newton_mjwarp EXPECTED_BACKEND=newton \
NUM_ENVS=256 MAX_ITERATIONS=2 \
RESULT_DIR="$PWD/output/T6-newton-gate" \
  ./scripts/run_isaac_lab_training.sh
```

Use a unique `RESULT_DIR` for every run. The harnesses refuse to overwrite
some nonempty result directories by design.

## 4. Reference Matrix

```bash
./scripts/run_gpu_physx_smoke.sh

NUM_ENVS=16384 MAX_ITERATIONS=250 \
SEEDS_CSV=20260720,20260721,20260722 \
  ./scripts/run_physx_soak.sh

./scripts/run_tiled_camera_progression.sh
./scripts/run_cosmos_writer_headless.sh
./scripts/run_cosmos_writer_streaming.sh

NUM_ENVS=4096 MAX_ITERATIONS=100 \
SEEDS_CSV=20260720,20260721,20260722 \
  ./scripts/run_backend_comparison.sh
```

The full matrix can run for hours and use substantial disk for caches, logs,
images, videos, telemetry, and checkpoints.

## 5. Visual Replays

The Full Streaming deployment can load the GPU PhysX and tiled-camera demos
through compose overrides:

```bash
docker compose --env-file deploy/.env -p isaacsim-tests \
  -f deploy/docker-compose.yml \
  -f deploy/docker-compose.visible-physx.yml \
  up -d --force-recreate
```

```bash
docker compose --env-file deploy/.env -p isaacsim-tests \
  -f deploy/docker-compose.yml \
  -f deploy/docker-compose.visible-tiled.yml \
  up -d --force-recreate
```

For the Newton policy replay, choose a checkpoint generated by T6:

```bash
PUBLIC_IP=<trusted-host-address> \
SOURCE_CHECKPOINT=/absolute/path/to/model_99.pt \
  ./scripts/start_newton_visual_replay.sh
```

Stop it with:

```bash
./scripts/stop_newton_visual_replay.sh
```

## 6. Publication Audit

Generated output is private by default. Before copying any evidence into Git:

```bash
./scripts/audit_public_tree.sh
git status --short
git diff --cached
```

Follow [privacy.md](privacy.md) for logs and browser media.
